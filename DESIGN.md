# 设计文档

## 1. 需求理解

比赛议题要求开发一款纯语音控制的绘图工具。用户不能使用鼠标或键盘完成绘图创作，只能通过语音指令驱动画布。

核心评价点包括：

1. 指令理解的准确性与容错性。
2. 语音到绘图操作的响应延迟。
3. 复杂指令的拆解与执行能力。
4. 作品完整度、交互流畅度和创新性。

## 2. 产品定位

本项目定位为一个语音驱动的矢量绘图白板。

它不把语音直接转换成不可编辑图片，而是把语音转换为可执行的绘图 DSL，再由 Canvas 渲染为可继续编辑的图形对象。

## 3. 用户故事

计划支持：

| 用户故事 | 当前状态 |
| --- | --- |
| 用户可以启动语音监听 | 已实现 |
| 用户可以语音创建基础图形 | 已实现 |
| 用户可以指定颜色、位置和大小 | 已实现 |
| 用户可以使用 A1-C3 坐标格定位对象 | 已实现 |
| 用户可以修改最近创建对象 | 已实现 |
| 用户可以撤销、重做、清空画布 | 已实现 |
| 用户可以创建组合对象 | 已实现 |
| 用户可以一句话生成多个对象 | 部分实现 |

| 用户可以导出 PNG / SVG | 待实现 |

## 4. 系统架构

```text
浏览器前端
  ├─ Web Speech API 语音识别
  ├─ 本地规则指令解析器
  ├─ 可选 DeepSeek 指令解析器
  ├─ 绘图 DSL
  ├─ 绘制步骤规划器
  ├─ 海龟式画笔状态
  ├─ Canvas 渲染器
  ├─ 对象状态管理
  ├─ 撤销 / 重做栈
  └─ 执行日志

Node 本地服务
  ├─ 静态资源服务
  ├─ /api/llm-status
  └─ /api/llm-command -> DeepSeek OpenAI-compatible Chat API
```

## 5. 绘图 DSL

项目使用结构化 JSON 表达绘图动作。

示例：

```json
{
  "actions": [
    {
      "type": "create_shape",
      "shape": "circle",
      "fill": "#60A5FA",
      "position": "center",
      "size": 120,
      "style": "cute_flat"
    }
  ]
}
```

当前支持的动作：

- `create_shape`
- `create_composite`
- `update_object`
- `resize_object`
- `move_object`
- `delete_object`
- `undo`
- `redo`
- `clear_canvas`
- `pen_down`
- `pen_up`
- `turtle_forward`
- `turtle_turn`
- `turtle_home`
- `turtle_color`
- `turtle_width`
- `set_grid`

规划型 DSL 可以额外带 `plan` 字段，用于解释复杂对象或画笔动作的拆解过程：

```json
{
  "plan": ["画一个圆形作为小猫的头", "在头顶画两个三角形耳朵", "画眼睛、鼻子和胡须"],
  "actions": [
    {"type": "create_shape", "shape": "circle", "fill": "#fb923c", "position": {"x": 0.5, "y": 0.36}, "size": 120, "label": "小猫头部"},
    {"type": "create_shape", "shape": "triangle", "fill": "#fb923c", "position": {"x": 0.43, "y": 0.25}, "size": 46, "rotation": -18, "label": "左耳"}
  ]
}
```

## 6. 指令理解策略

当前 MVP 采用“本地规则优先 + DeepSeek 可选增强”的混合解析策略，优先保证低延迟、可控性和 Demo 稳定性。

解析流程：

```text
中文语音文本
  -> 清理空格和标点
  -> 判断系统命令
  -> 拆分多段指令
  -> 识别颜色、位置、大小、坐标格、对象类型
  -> 生成绘图 DSL
  -> 执行动作
```

复杂口令会通过本地 Node 服务调用 DeepSeek，让模型输出 JSON；如果未配置 Key、网络失败或模型返回内容不可执行，则自动回退到本地规则解析。

对于“画一只小猫”“画一辆小汽车”这类完整物体，当前先使用本地规划器拆解为步骤和基础图形。执行时绘图笔游标会移动到每个部件位置，再逐个落下图形，避免用户感觉系统是在粘贴一张现成图片。后续可以把规划器扩展为 LLM 生成步骤、前端白名单校验执行的流程。

为降低语音描述位置的难度，画布默认显示 A1-C3 九宫格辅助线。用户既可以说“左上角、右边、下面”等自然方位，也可以说“在 B2 画一个圆”“把它移到 C3”。坐标格只作为构图辅助，不改变画布对象模型；画面上不直接显示 A1/B2 标签，避免干扰作品观感。

## 7. Prompt 设计

DeepSeek 系统 Prompt 采用以下原则：

```text
你是一个语音绘图指令解析器。你的任务是把用户的中文语音指令转换为绘图操作 JSON。

规则：
1. 只能输出 JSON，不要输出解释。
2. 如果一句话包含多个绘图动作，请拆分成 actions 数组。
3. 优先使用系统支持的基础图形、组合对象和样式参数。
4. 如果用户说“它”“刚才那个”“这个”，优先引用最近创建或最近选中的对象。
5. 如果目标对象无法确定，返回 clarification 字段，向用户提出一个简短问题。
6. 如果用户要求复杂对象，请先判断是否命中模板；如果没有模板，则用基础图形和部件组合成简化版本。
7. 不要输出无法执行的自由描述。
8. 所有图形默认使用 cute_flat 风格。
```

实现约束：

1. API Key 只保存在本地服务端环境变量或 `.env`，不会暴露给浏览器。
2. DeepSeek 输出必须经过前端 DSL 白名单过滤，只允许执行已支持动作。
3. 默认模型为 `deepseek-v4-flash`，如需更强理解可配置为 `deepseek-v4-pro`。
4. 默认关闭 DeepSeek thinking mode，降低延迟并减少 JSON 输出不稳定性。

## 8. 审美系统

项目采用儿童绘本式扁平矢量贴纸风。

视觉规则：

- 深色统一描边
- 明亮柔和配色
- 圆角和柔和曲线
- 少量阴影
- 不追求真实照片感，追求清晰、可爱、可编辑

## 9. 组合对象策略

当前支持：

- 太阳
- 云
- 树
- 房子
- 花
- 小女孩

组合对象不是图片贴图，而是由 Canvas 路径和基础图形绘制，后续可以继续被移动、缩放和修改颜色。

规划对象与组合对象的区别是：组合对象是一个可移动缩放的整体模板；规划对象会拆成多个基础图形部件，右侧“绘制步骤”显示它如何被画出来。这样既保留美观的默认效果，也给后续“模型自己拆步骤画图”留下空间。

规划对象执行时不会直接把完整图像贴到画布上，而是按动作列表逐步执行。画布上会出现绘图笔游标，它移动到下一个部件位置后再绘制该部件，用可见过程表达“AI 正在画”。

## 10. 海龟式画笔

为了解决“只会放库里的形状”的局限，项目新增了海龟式画笔状态：

- 画笔位置：使用画布归一化坐标保存。
- 画笔方向：0 度向右，右转为正角度。
- 落笔状态：落笔时移动会生成线条，抬笔时只移动光标。
- 画笔属性：支持颜色和线宽。
- 路径预设：支持“用画笔画正方形 / 三角形 / 房子轮廓”，自动拆成落笔、前进、转向、抬笔动作。

典型口令：

```text
落笔
向前走一百
向右转九十度
再向前走六十
抬笔
回到中心
用画笔画一个正方形
用画笔画一个三角形
```

这个能力让用户可以逐步“教”系统画路径，也为 LLM 把复杂轮廓拆成路径动作提供基础。

## 11. 延迟优化

当前版本使用本地规则解析，不依赖网络模型，因此常用指令响应很快。

接入 LLM 后使用混合策略：

1. 简单命令本地解析。
2. 复杂指令调用 DeepSeek。
3. LLM 输出必须经过 DSL 白名单过滤。
4. 解析失败时降级为澄清问题或简化版本。

## 12. 未完成部分与原因

| 功能 | 原因 |
| --- | --- |
| JSON Schema 校验 | 当前先用白名单过滤，后续再抽成显式 Schema |
| PNG / SVG 导出 | MVP 优先验证语音绘图闭环 |
| 更精细人物模板 | 人物审美要求高，需要更多视觉打磨 |
| 复杂场景自动布局 | 需要碰撞检测和构图算法 |
| 语音澄清对话 | 需要更完整的对话状态管理 |
| 通用物体规划 | 当前只做小猫、小汽车初版，后续再接 LLM 生成步骤 |

## 13. 后续优化方向

1. 接入 LLM 解析复杂自然语言。
2. 增加更多组合对象和可参数化部件。
3. 支持导出作品。
4. 支持语音确认和语音澄清。
5. 增加对象选择能力，例如“选择左边的树”。
6. 增加自动构图，使复杂场景更美观。
7. 让 LLM 输出“绘制步骤 + 海龟路径 + 基础图形”，逐步减少硬编码模板依赖。
