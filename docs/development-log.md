# 持续交付记录

本文档用于记录 72 小时比赛开发过程中的阶段性工作、提交和 PR。后续所有功能开发都应通过功能分支和 Pull Request 合入 `main`。

## 1. 提交原则

1. `main` 分支保持可运行状态。
2. 每个明确功能点单独创建分支。
3. 每个分支至少包含一个有意义的 commit。
4. 每个阶段通过 PR 记录设计、实现和验证内容。
5. 不在最后一天一次性导入大段代码。

## 2. 分支命名

| 类型 | 格式 | 示例 |
| --- | --- | --- |
| 功能 | `feature/<name>` | `feature/llm-dsl-parser` |
| 文档 | `docs/<name>` | `docs/continuous-delivery-plan` |
| 修复 | `fix/<name>` | `fix/undo-state` |
| 视觉打磨 | `polish/<name>` | `polish/composite-templates` |
